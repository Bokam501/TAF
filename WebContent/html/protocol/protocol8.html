Validation Protocol

Test Description:
The purpose of this protocol is to demonstrate with ZemPro that the application clears calculator entries and output results from Dose Initiation and Dose Titration Dose modules when the User selects or the mobile device turns into background or sleep state.

Acceptance Criteria:
Acceptance criteria are defined as successful completion of the steps identified below with results matching the description of Expected Results.

Prerequisites:
  • A qualified mobile device is provided to the tester.
  • Tester must have basic experience with iPhone, iPad and iPod Touch operationss
  • Tester must use the workstation clock and time format hh:mm for timing actions.


Notes to Tester:
  • The “Comments” field should be left blank unless the step requires additional explanation
  • Turn off Internet Service
    • Tap Settings
    • Turn Airplane Mode ON
  • Auto-lock setting for 2 minutes = Device is in sleep mode when left idle for minimum 2 minutes. (The Auto-lock feature on the device allows setting the amount of time that elapses before the iPhone automatically locks or turns off the display.)
  • To set auto-lock option
    • tap the settings icon on the device screen
    • tap on General
    • tap on the Auto-lock option
    • tap on 2 Minutes
  • Application is in background state = the device clock timer pop up window is presented.
  • Close the application from background state = One click on the device home button to close the task bar.
  • Exit application = Press the mobile device home button.
  • To set Timer option
    • tap the Clock icon on the device screen
    • tap on Timer
    • tap on on 1 min and then Start
  • Abbreviations: SRS = software requirements specification
    EULA = End User License Agreement
    ISI = Important Safety Information
    PI = Prescribing Information
    SR = Software requirement
