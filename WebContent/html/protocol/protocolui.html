Purpose:
The purpose of this protocol is to confirm the labeling standards throughout the application meet the product requirements as specified below, per DHF-011-0010 Trace Matrix.

PR#Product Requirement
PR1The app shall provide electronic access to the US Prescribing information.
PR2The app shall provide electronic access to the Important Safety Information.
PR3The app shall provide electronic access to the End User License Agreement.
PR11The app user interface shall incorporate signs, symbols, and markings, and Software User Interface guidelines from ANSI/AAMI HE75:2009.
PR12The app shall incorporate Zemplar branding and AbbVie brand style guidelines.
PR13The app shall be provided in the English language.
PR18The app shall display the End User License Agreement and require acceptance of terms and conditions the first time after installation and after each software update.
PR19The app shall require confirmation of health care provider status.
PR24The app shall provide Instructions for Use.

Acceptance Criteria:
Acceptance criteria are defined as successful completion of the steps identified below with results matching the description of Expected Results. Note that text flow or line break differences are not considered failure and need not be noted in results.

Prerequisites:
  • A qualified mobile device with a new installation of ZemPro is provided to the tester.
  • Approved copies of the following are provided to the tester:
    • DHF-011-0013(GUI)
    • Zemplar Injection Important Safety Information(ISI)
    • Zemplar (paricalcitol) Injection Prescribing Information(PI)
    • ZemPro User Guide (PIFU-0010)
  • Tester must have basic experience with iPhone operations


Notes to Tester:
  • For steps requiring data entry in multiple fields using the keypad, the ‘Next’ button will navigate to the next entry field. ‘Previous’ button will navigate to the previous entry field. The ‘Done’ button will close the keypad.
  • UI references in parentheses in the Actual Results column are for traceability and reference to assist the tester in navigating the GUI requirement document as needed.
  • The “Comments” field should be left blank unless the step requires additional explanation
  • In Pass/Fail column the following nomenclature is acceptable:
    • “P”= Pass
    • “F”= Fail
    • N/A = (not applicable). N/A is prepopulated in each step used to describe navigation or steps not performed.
  • Turn off Internet Service
    • Tap Settings
    • Turn Airplane Mode ON
  • Abbreviations: PR = Product requirement
    EULA = End User License Agreement
    ISI = Important Safety Information
    PI = Prescribing Information
    GUI= Graphical User Interface
  • Screen elements are evaluated for the following:
    • Artwork – non-text graphic elements, stylized text, and color schemes.
    • Layout - review the general screen arrangement and colors of the following (ignore spacing and status bar found at the top of some images in Expected Results)
      • Text
      • Artwork including Icons and symbols
      • Buttons
      • Entry Fields
    • Text –content of text present and of specific text cited in Expected results.
    • Buttons – shape, color and labeling.
  • Screen shots in Expected Results are for reference purposes of layout only. Refer to approved GUI Requirements document for expected results. Note navigational buttons/text (Home, Clear, Close, Back), keypad and notifications will render differently between iOS versions.
