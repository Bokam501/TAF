Test Case Execution

Overview

This page allows you to execute the test cases allocate to you on the selected date. You can select a date and work package, such that the test cases allocated are displayed in the Test Cases pane. You can execute the test case, and record the results in the Results tab, and defects in the Defect tab.

 

Filters

Field

Description

Planned Execution Date

Select a date to list allocated work packages.

Work Package

Select a work package to list the allocated test cases for the selected date.

 

To access this page, click My Tasks > Allocations.

  • Test Case
  • Test Steps
  • Result
  • Defect

Test Case

This tab displays details of the test case for execution such as test case ID and name, description, priority, execution type, testing environments and status.

 

Test Steps

This tab displays name of the test case, test step input, expected output of the test case, observed output for the test case, and result of the test case either it is passed or failed.

 

Result

This tab allows you to enter the test case result based on the execution.

Fields

Field

Description

Preconditions

Displays the preconditions required to perform the test case.

Input

Displays the input to be entered in the test case execution.

Expected output

Displays the expected output from the test case execution.

Observed Output

Enter the observed output of the test case execution.

Comments

Enter comments if any to the reviewer.

Actual Execution Date

Select the actual test case execution date.

Result

Select one of the following options:

  • Pass
  • Fail
  • No Run
  • Blocked

Executed

Select Yes or No to mark whether the test case is executed or not.

Actual Shift

Select the actual shift from the drop-down list.

Upload Evidence

Click the Upload button or drag and drop the evidence into this field.

Evidence Uploaded

Displays whether the evidence is uploaded or not.

 

 

Defect

Fields

Field

Description

Bug Title

The title of the bug.

Description

Severity

  • Blocker
  • Normal

Filing Status

Select one of the following options:

  • New
  • ReferBack
  • Reviewed
  • Approved
  • Closed

Remarks

Enter remarks if any.

Creation Time

Displays the date on which the bug is created.
