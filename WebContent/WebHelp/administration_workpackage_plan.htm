Configure Work Package

Overview

This page is used to configure the work package with test suites/case, testing environments, and devices, and allocate the test case to resources to execute them on a shift of a specific date. Also, you can view the results of the test case execution, and defects reports registered by the resources.

  1.  

To access this page, login as test manager/test lead, and click Plan > Configure Work Package. Click the following tab page for more information:

  • Work Package Summary
  • Plan
  • Allocate Test Cases
  • Raise Resource Demand*
  • Reserve Resources*
  • Results
  • Defects

Work Package Summary

The Work Package Summary tab displays the Work package Details and Test Case Execution Status panes.

 

Work Package Details pane

The Work Package Details pane displays the product name, build, planned/actual start date and end date for completing the work package, and overall status of the work package.

 

Test Case Execution Status pane

The Test Case Execution Status pane displays the test case executed date, shift, count of test cases completed, defects raised for the test cases, and number of test cases approved and reviewed.

 

 

Plan

The Plan tab displays the list of available test suites and test cases for this product. You can select the test case, and map it to testing environments and devices. Based on the test case and testing environment combination, the test cases in the Allocate Test Cases tab will be generated.

Tabs

In this tab, you can map Features, Test suites or Test cases to the work package. For example, a feature can be mapped to a work package to execute the test cases available for the feature. Also, you can add similar test cases or test suites which are not available for the feature.

Allocate Test Cases

The Allocate Test Cases tab displays the list of test cases and its testing environment combination. You can allocate these test cases to resources to execute them on a specific shift of a date. This tab also displays the completion status of the test cases. To perform bulk allocation, select corresponding features, test suites, or test cases, and select the planned execution date, shift, tester name, test lead name, and then click Save.

 

 

Raise Resource Demand

The Raise Resource Demand tab allows to raise resource demand for a shift on a specific date. You can click the shift that corresponds to a date, and add the type of resource and number of resource required.

 

 

 

Reserve Resources

In the Reserve Resource tab, you can reserve the resource available for reservation, view the resource demand raised, and gap to be filled. This tab is available only for the standard engagement mode.

 

 

 

Results

In the Results tab, you can view the test case results.

 

 

Defects

In the Defects tab, you can view defects raised by resources for the test cases.
