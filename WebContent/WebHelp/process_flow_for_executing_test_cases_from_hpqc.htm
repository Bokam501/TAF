Process Flow for Executing Test Cases from HPQC

This section describes how to configure HPQC, download Test Cases from HPQC, and execute them.

Prerequisites

Make sure the following things are in place, before you configure, create and execute the sample test run plan:

  • Set up a test engagement. For more information, Create a test engagement.
  • Create a product, and its version and build. For more information, Create a product.
  • Set up environments and environment combination. For more information, Manage Product Attributes.
  • Map available active devices to a product. For more information, Manage Product Attributes.

Configure HPQC connection

You can download the test cases and test suites from the HPQC, and also upload the execution results of the test cases back to the HPQC. Before performing the uploading or downloading operation, you must establish the connection between the HPQC and iLCM. For example, the HPQC has a domain i.e. default, and a project i.e. product1.

 

To establish connection between HPQC and iLCM, follow these steps:

  1. Click Setup > Product Management. On the Products pane, select the Engagement > Product1.
  2. On the Manage Products page, click the Tool Integration tab.
  3. On the Test Management System tab, click the Add button. As a result, the Add page is displayed.
  4. In the Title field, enter the test management system name i.e. HPQC, and in the Description field, enter the description for the test management system.
  5. In the Test System drop-down list, select the test management system as HPQC.
  6. In the Version field, enter the test management system version number, and in the Connection Uri field, enter the url of the test management system say “http://10.108.57.17:8080/qcbin”.
  7. Enter the username, password, and property 1 i.e. “default”, and then click the Save button.

To add the entity, follow these steps:

  1. On the Test Management System tab, click the Test Management System Mapping () icon, and then on the Test Management System Mapping pane, click the Add button.
  2. In the Mapping Entity drop-down list, select one of the following options:
    • Product (Select this option to view the list of  products available in the HPQC)
    • Product Version
    • Test Suite
  3. In the Entity Name drop-down list, select the entity name i.e. Product 1. The test case for the selected product name will be imported from the test management system.
  4. In the Mapping Value field, enter the mapping value for the product name i.e. Product. Here the Mapping value indicates the project name in HPQC HPQC system.

Set up Test Suite with Scripts

Test run plan executes test scripts, which are mapped in a test suite.

To configure the test scripts for the test suite, follow these steps:

 

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and on the Test Suites tab, click the Add button.
  3. Enter the test suite name, and select the script type such as JUNIT, PYTHON, and TESTNG, from the drop-down list.
  4. Enter the script file location of the testing script (e.g. file:///D:/iLCM/TestScripts/iLCMBDD.zip) in the Script File Location field, and enter the test suite code for the test suite in the Suite Code field.
  5. Select Execution Type such as Mixed, Hybrid, Automated, and Manual, from the drop-down list, and then click the Save button.

—OR—

To download the test scripts available in the HPQC location, follow these scripts:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product, and on the Test Cases tab, click the Import Test Management System button.
  3. As a result, the test scripts and test suites available in the HPQC are downloaded in the iLCM.

 

Create Test Run Plan

Test run plan is a combination of Test beds and test scripts, used to create a work package. Test run plan can be triggered to create work package and execute the test cases either manually or automatically. To create a test run plan, follow the steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and click the Test Run Plan tab.
  3. Click the Add New Test Run Plan button, and on the Create Test Run Plan page, select the Execution Type, and Test Tool Master.
  4. Click the Submit button to create a test run plan, and then click the Continue button.
  5. In the Select Test Suites page, on the Test Suite tab, select the test suites, and click the Continue button.
  6. On the Select Test Beds page, map the target devices corresponding to the test environment combination, and then click the Continue button.
  7. The test suites and test cases are listed in the Test Beds Distribution page.
  8. Click the Continue button, and on the Execute page, select the Product Build, and then click the Execute button.

Execute Test Run Plan

To execute a test run plan, follow the steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and click the Test Run Plan tab.
  3. On the Test Run Plan(s) page, click the Execute Test Run Plan ( ) icon. As a result, the work package for the corresponding Test Run Plan will be created.
    Note: If the Test Run Plan Type is automated, then run the Terminal.Jar file, and check whether the handshake is done between the server and the terminal.
  4. If the Test Run Plan is created for automated test case execution, then the test cases will be executed after the work package is created successfully.

Upload Test Results to HPQC

After executing the test cases, you can upload the test cases execution result back to the HPQC. To upload the test results, follow these steps:

  1. Click Plan > Test > Work Packages.
  2. On the Work Packages pane, expand the tree and select the Product > Build > Work Package.
  3. On the Plan page, click the Results tab, and then click the Test Run Job tab.
  4. On the Test Run Job Result Summary pane, click the Export Results button to upload the test execution result to the HPQC.
