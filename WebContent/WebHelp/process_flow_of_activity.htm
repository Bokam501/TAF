Process Flow of Activity LifeCycle

This section describes how to create and execute a task in an activity workpackage.

 

Prerequisites

Make sure the following things are in place, before processing the activity workpackage:

  • Set up a test engagement. For more information, Create a test engagement.
  • Set activity types and associate weightage for the activity types. For more information, Set Up Activity Type .
  • Create a product, and its version and build. For more information, Create a product.
  • Set up features, and environment combination (device combinations). For more information, Manage Product Attributes.
  • Set up dimension management along with the status and competency management. For more information, Set up Dimension Management.

Create Activity Work Package

Activity work package contains group of activities and tasks to be performed. The work package is schedule to complete within a time frame of a project. Each work package is associated with a owner, and priority for the completion of it.

 

To create an activity work package, follow these steps:

  1. Click the Plan > Activity Management > Activity Work Package.
  2. On the Products pane, select the Engagement > Version > Build.
  3. On the Activity Work Package page, click the Add button.
  4. Select the Build from the drop-down list, and enter the unique activity package name.
  5. Select the owner who is taking responsibility for the work package from the drop-down list.
  6. Optional: Select the competency from the drop-down list.
  7. Select the priority, planned starting date, and planned ending date.
  8. Click Save to create the activity work package.

Create Activity for Work Package

An activity is a set of predefined tasks to be completed within a specific period. The activity might be changed according to the change requests given by the clients. Also, the clarifications raised for each task in the activity can be mapped against each activity.

 

To create activity, follow these steps:   

  1. Click the Plan > Activity Management > Activity.
  2. On the Products pane, select the Engagement > Version > Build > Activity Work Package.
  3. On the Activity page, click the Add button, and enter the unique activity name.
  4. Select the requirement type from the drop-down list.
    Note: The requirement drop-down list is defined in the Product Management page as Features.
  5. Select the activity type from the drop-down list.
    Note: Activity type is defined in the Admin > Manage Customer > Activity Type(s).
  6. Optional: Enter the tracker number for the activity, if required, and select the execution type of the activity as Manual, Automated, or Both.
  7. Select the assignee and reviewer name.
  8. Select execution priority for the activity, and select the planned starting date and planned ending date.
  9. Click Save to create the activity.

Map Change Request and Clarifications

After creating the activity, you can map the change request and clarifications received for the activity. To create and map the change request, follow these steps:

  1. In the Activity Summary page, click this () icon under Change Request field, and click Add to add a new change request received.
  2. Enter the change request and description, and select execution priority, status, and owner name.
  3. Click Save to create the change request.
  4. Click this icon () under Change Request Mapping field, and drag and drop the change request to map them with the activity.
  5. Also, you can click this () icon under Clarifications field, and click Add to add the clarifications.
  6. Enter a title and description for clarification, and select execution priority, status, and owner name.
  7. Click Save to create the clarifications.
  8. Click this icon () under Clarifications field, and drag and drop the clarifications to map them with the activity.

Create Task for Activity

Task can be sub category of an activity to be performed and reviewed.

 

To create task, follow these steps:   

  1. Click the Plan > Activity Management > Activity Task.
  2. On the Products pane, select the Engagement > Version > Build > Activity Work Package.
  3. On the Activity Task page, click the Add button, and enter the unique activity name.
  4. Select the task type, category, assignee and reviewer from the drop-down list.
  5. Select the execution priority, primary and secondary status from the drop-down list.
    Note: The primary and secondary status are defined in the Status Management page.
  6. Select the planned starting and ending dates on which the task is to be completed.
  7. Optional: Enter the planned task size and planned effort.
  8. Click Save to create the task.

Execute assigned task

Engineer can execute the assigned task or activity in his login. To execute the assigned task, follow these steps:

  1. Click the My Activities > Activities > My Activities. As a result, the Activities list is displayed with the activity details of all work package.
  2. On the Activity pane, select the Activities. As a result, on the Activity Summary tab, the activity details is displayed.
  3. Click the Activity Task tab to view the list of assigned task.
  4. Click the Edit Status () icon to change the status of the task after completing the task.
  5. In the Edit Status page, select the primary and secondary status.
  6. In the Previous Status Effort (in hr) field, enter the time spent for the task in hours.
  7. Optional: Enter comments regarding the task, and click Save.
  8. You can add additional tasks apart from the assigned tasks. For more information, refer Create Task for Activity.
