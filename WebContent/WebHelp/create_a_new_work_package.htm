Create a New Work Package

Work package for a product can be created manually, automatically, or by cloning the existing work package. The test manager or the test lead of the product can manually create a work package, and plan the completion date of the work package. The test manager can create test run plans in the product management page to automatically create the work package, which can be executed automatically or manually. Also, the test manager can clone specific parameters of the existing work packages to create a new work package based on the selected parameters.

Create a work package

To create a work package for a product, follow these steps:

  1. Click Plan > Test > Create New Work Package. In the Create Work Package page, on the Work Packages pane, expand the Product >  Version > Build.
  2. On the Create Work Package pane, enter the details such as work package name, description for the work package, and start date and end date for completing the work package.
  3. In the Advanced field, select the Create New option to create a new work package.
  4. Select either Manual or Automated option from the Work Package Type drop-down list.
  5. Click the Create button to create a work package.
    Note: You will be navigate to the Work Package Execution page.

Create a work package using cloning

To create a work package for a product, follow these steps:

  1. Click Plan > Test > Create New Work Package. In the Create Work Package page, on the Work Packages pane, expand the Product >  Version > Build.
  2. On the Create Work Package pane, enter the details such as work package name, description for the work package, and start date and end date for completing the work package.
  3. Select either Manual or Automated option from the Work Package Type drop-down list.
  4. In the Advanced field, select the Copy Existing option to copy parameters such as Environments, Planned Execution Date, Shift, Resource Demand, Test Lead, and Tester, from the existing work package.
  5. Select the existing work package from the Work Packages drop-down list.
  6. Select the Environments, Execution Date, Shift, Resource Demand, Test Lead, and Tester check boxes. The Features, Test Suites, and Test Cases check boxes are selected by default.
    Note: These are available only when the Copy Existing option is selected.
  7. Click the Create button to create a work package.
    Note: The Create Work Package will be navigate to the Work Package Execution page.
