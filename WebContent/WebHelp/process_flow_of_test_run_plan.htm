Process Flow of a Sample Test Run Plan

This section describes how to configure, create and run a sample test run plan.

Prerequisites

Make sure the following things are in place, before you configure, create and execute the sample test run plan:

  • Set up a test engagement. For more information, Create a test engagement.
  • Set up vendor account for the resource. For more information, Create vendor accounts.
  • Set up customer account for the product. For more information, Create customer account.
  • Create user profile for the test engagment. For more information, Create a resource profile.
  • Create a product, and its version and build. For more information, Create a product.
  • Set up environments and environment combination. For more information, Manage Product Attributes.
  • Map available active devices to a product. For more information, Manage Product Attributes.

Set up a test suite with the scripts

Test run plan is executed using the test scripts, which are mapped or file location is configured in a test suite. To create a test suite for automated testing (Test Run Plan), follow these steps:

    1. Click Setup > Product Management.
    2. On the Products pane, select an Engagement > Product > Version, and on the Test Suites tab, click the Add button.
    3. Enter the test suite name, and select the script type such as JUNIT, PYTHON, and TESTNG, from the drop-down list.
    4. Enter the script file location of the testing script (e.g. file:///D:/iLCM/TestScripts/iLCMBDD.zip) in the Script File Location field, and enter the test suite code for the test suite in the Suite Code field.
    5. Select Execution Type such as Mixed, Hybrid, Automated, and Manual, from the drop-down list, and then click the Save button.

Create Test Run Plan

Test run plan is a combination of Test beds and test scripts, used to create a work package. Test run plan can be triggered to create work package and execute the test cases either manually or automatically. To create a test run plan, follow the steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and click the Test Run Plan tab.
  3. Click the Add New Test Run Plan button, and on the Create Test Run Plan page, select the Execution Type, and Test Tool Master.
  4. Click the Submit button to create a test run plan, and then click the Continue button.
  5. In the Select Test Suites page, on the Test Suite tab, select the test suites, and click the Continue button.
  6. On the Select Test Beds page, map the target devices corresponding to the test environment combination, and then click the Continue button.
  7. The test suites and test cases are listed in the Test Beds Distribution page.
  8. Click the Continue button, and on the Execute page, select the Product Build, and then click the Execute button.

Execute Test Run Plan

To execute a test run plan, follow the steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and click the Test Run Plan tab.
  3. On the Test Run Plan(s) page, click the Execute Test Run Plan () icon. As a result, the work package for the corresponding Test Run Plan will be created.
    Note: If the Test Run Plan Type is automated, then run the Terminal.Jar file, and check whether the handshake is done between the server and the terminal.
  4. If the Test Run Plan is created for automated test case execution, then the test cases will be executed after the work package is created successfully.
