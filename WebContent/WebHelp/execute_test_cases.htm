Execute allocated test cases

The tester or test lead can view the test case allocation in the Allocations page. The resource must select a work package and a date to view the allocated test cases to be executed. After completing the test cases, the resource can log the results along with the evidence document. Also, he can log the bug details for the test case.

 

Execute allocated test cases for the shift

To execute the work package, follow these steps:

  1. Click My Tasks > Allocations.

  2. On the Test Case Execution page, in the Planned Execution Date field, select a date, and then in the Work Package drop-down list, select a work package.

  3. On the Test Cases pane, select one of the test cases. The Test Case Execution page is update with the Test Case, Test Steps, Result, and Defect panes. By default, the Test case section is expanded. Also, a timer is set to record the execution time of the test case with the Resume, Pause, and Restart buttons.

    1. The Test Case pane displays details of the test cases such as test case name, description, environments, test case priority, and test case execution type.

    2. The Test Steps pane displays Name, Description, TestStep Input, Expected Output, Observed Output, Result, and Comments.

    3. The Result pane allows the resource to enter result of the test cases such as observed output, comments, executed or not, state of the test case, evidence, and actual execution date.

    4. The Defect pane allows the resource to enter the bug details such as bug title, description, environment, and severity.

  4. On the Result pane, enter the observed output and comments.

  5. Select the actual execution date and the actual shift on which the test case is executed.

  6. In the Result field, select the Pass, Fail, No Run, or Blocked option based on the test case outcome.

  7. In the Executed field, select either Yes or No to confirm whether the test case is executed.

  8. Click the Upload button to upload an evidence document or file, or drag and drop the evidence document or file.

  9. On the Defect pane, click the Add button to add the defect details.

  10. Enter the bug title, description, system name, system bug ID, and file bug.

  11. Select the Severity, and Filing Status, and then click the Save button to save the changes made.
