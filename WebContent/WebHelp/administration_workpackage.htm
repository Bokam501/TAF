New Work Package

Overview

This page allows you to create a new work package with the planned starting and ending dates. Work package is a combination of a group of test cases or test suites, and testing environment, which is executed by a test lead or tester on a specific shift. These work package executions are later verified by the test managers.

 

To access this page, login as Test Manager or Test Lead, and click Plan > New Work Package.

Note: Make sure, on the Work Packages pane, you have selected the Product > Version > Build.

Fields

Field

Description

Work Package Name

Name of the work package.

Advanced

Select on the following options:

  • Create New — select this option to create a new work package.

  • Copy Existing — select this option to create a new work package based on parameters such as environments, shift, and resources, available in the existing work package.

Description

Brief description about the work package.

Work Package Type

Select the following option:

  • Manual — select this option to perform a manual testing of the product.

  • Automated — select this option to perform automated testing of the product.

Planned Start Date

Select the planned starting date for the execution of the work package.

Planned End Date

Select the planned ending date for the execution of the work package.

Work Packages

Select a corresponding work package to copy the parameters of it for creating a new work package.

Note: This field is available only when the Copy Existing option is selected.

Clone

Select the following options to copy the corresponding parameter of the existing work package to the new work package:

  • Environments

  • Planned Execution Date

  • Shift

  • Resource Demand

  • Test Lead

  • Tester

Note: This field is available only when the Copy Existing option is selected.

 

Buttons

Field

Description

Create

Click this button to create a new work package.
