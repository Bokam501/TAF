Create a Test Run Plan

A test manager can define a test run plan, and configure devices, environment combinations, and test suites to create a work package. The work package can be executed either manually or automatically.

 

Workflow diagram

The following illustration shows how to create and execute test run plan.

 

Prerequisites

  1. The terminal from the which the web application to be tested, is active.
  2. The URL of the web application to be tested, must be included in the Web Application URL field of the product version. For more information, see Manage Product Attributes.
  3. The file location of the testing script must be included in the Script File Location field of the Test Suite, which is mapped with the test run plan.
    Note: The file location of the testing script must be in the following format: (file:///d:/TestScripts/file_name.zip)

Create a test run plan

To create a test run plan, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and click the Test Run Plan tab.
  3. Click the Add New Test Run Plan button, and on the Create Test Run Plan page, select the Execution Type, Test Run Schedule, and Test Tool Master.
  4. Enter the Cron expression to schedule and auto execute the test run plans. Also, you can set up the cron scheduler after creating the test run plan.

    1. Click the Test Run Plan Schedule icon, and select Minutes, Hours, Daily, Weekly, Monthly, or Yearly tabs.

    2. Set the corresponding schedule parameters, and click Save to create the cron expression and auto execute the test run plan in the schedule.

  5. Click the Submit button to create a test run plan, and then click the Continue button.
  6. In the Select Test Script page, select the features, test cases, or test suites, and click Continue.
  7. On the Select Test Beds page, map the target devices corresponding to the test environment combination, and then click Continue. The test suites and test cases are listed in the Test Beds Distribution page.
  8. Click Continue, and on the Execute page, select the Product Build, and then click Execute.
    Note: Make sure that the iLCM Terminal is launched and the connection is established between the server and the terminal.
  9. The work package with test cases and test bed details are generated along with the execution results. TAF can create screenshots and evidence for the executed test run plan.

Execute an automated test run plan

To execute an automated test run plan, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and click the Test Run Plan tab.
  3. On the Test Run Plan(s) page, click the Execute Test Run Plan () icon. As a result, the work package for the corresponding Test Run Plan will be created.
  4. If the Test Run Plan is created for automated test case execution, then the test cases will be executed after the work package is created successfully.
