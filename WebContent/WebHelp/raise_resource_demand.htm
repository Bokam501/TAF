Raise Resource Demand

The test manager or test lead plans resources allocation for work package on a specific shift of the day, and forecasts the resource in demand to complete the work package. Based on the analysis, the test manager or test lead raises the resource demand for that specific shift.

 

Raise resource demand for the work package

To raise the resource demand for the work package, follow the steps below:

  1. Click Plan > Configure Work Package. On the Work Packages pane, expand the product > build, and then select the work package.
  2. On the Work Package Execution page, on the Raise Resource Demand tab, on the Create/Edit resource demand for executing test cases pane, click the cell corresponding to the shift and date. As a result, the Raise Resource Demand page is displayed.
  3. Click the Add button, and select the user role and number of resources required.
  4. Click the Save button to raise the resource demand.
