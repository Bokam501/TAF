Manage Dimensions

Dimension management includes status category and competency management. Competency management defines the sub division of the engagement, in which it can be mapped and management across multiple products. Status category defines the number of status involved in an activity work flow, where the status can be mapped to one or more products.

 

Prerequisites

Make sure the following things are in place to manage dimensions.

  1. Set up an engagement. For more information, Create a Test Engagement.
  2. Create user profiles for the engagement. For more information, Product Team Users.
  3. Create a product. For more information, Create a Product.htm.

Dimension Management

You can create the dimensions such as status category and competency in the Dimension Management page. To create dimensions, follow these steps:

  1. Click Setup > Dimension Management, and click Add.
  2. In the Add page, enter name and description of the dimension.
  3. In the Type field, select either Competency or Status to map the dimension with its type.
  4. If required, select parent node dimension from the drop-down list, and then click Save.

Status Management

You can define the number of status of a workflow in the status management. To define the status category of a workflow, follow these steps:

  1. Click the Setup > Status Management. On the Manage Status pane, select the status category.
  2. As a result, the Status Summary tab displays the details of the status category such as number of primary status, and secondary status.
  3. Click the Primary Status tab, and click Add to add the primary status.
  4. In the Add page, enter the name and description of the primary status.
  5. Select one of the default status category in-which the primary status comes in.
  6. Click Save to create the primary status.

To create the secondary status for the primary status, follow these steps:

  1. In the Status Management page, click the Secondary Status tab, and click Add.
  2. Enter the name and description of the secondary status, and then click Save.
  3. After creating the secondary status, click the Primary Status tab.
  4. Click the () icon of a primary status, and drag and drop the secondary status from the available secondary status list to mapped secondary status list.

 

Competency Management

You can define the team members and products for competency in the competency management. To define the competency, follow these steps:

  1. Click the Setup > Competency Management. On the Manage Competency pane, select the competency.
  2. As a result, the Competency Summary tab displays the competency details such as number of products and member available for the competency.
  3. Click the Team tab, and click Add to add the team members to the competency.
  4. In the Add page, select the team member, and his beginning and ending working dates.
  5. Click Save to allocate the team member for the competency in the specific dates.
  6. Click the Products tab, and click the Product Members () icon of a product.
    Note: You must map the competency to a product in the Product Management page.
  7. Click Add to add the team members for the products.
  8. Select the user name, role, from and to date in which the team member will be available for the product.
  9. Click Save to map the user for the product.
