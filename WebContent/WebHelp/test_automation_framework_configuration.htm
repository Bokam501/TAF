Configure Test Automation Framework

Test automation framework (TAF) automatically runs the test scripts in a test run plan on a mobile device or server connected using the TAF terminal. TAF terminal initiates the connection between the server and client, and creates a work package with the test cases and environments based on the Test Run Plan. It also executes the test scripts and compares the expected output with the observed output, and logs the results.

 

Prerequisites

Make sure the following are in the place:

  1. Latest version of Java is installed in the system.
  2. The TAF Config file is located in the "%userprofile%/TAF" location.
  3. The latest TAF Terminal.jar file is available in the Terminal folder.
  4. The test scripts files are available in the Test Scripts folder. The location of the Test Scripts folder is specified in the Web Application URL field of Product Version.

Configure TAF properties

To configure the TAF Config file, follow these steps:

  1. Right click the TAF.config file, and select the Open with option in the short menu.
  2. Select the Notepad option in the Open with dialog box, and click the OK button.
  3. Specify the server url in the SERVER_URL field. For example, http\://localhost\:8080/iLCM.
  4. Specify the library folder locations for the testing application such as SELENDROID, TESTNG, SEETEST, and JUNIT.
  5. If required, specify the Heart beat frequency for the server and terminal connection in the HEARTBEAT_FREQ field.
  6. You can also set the Log file location in the TERMINAL_LOG_FILE field.

Run TAF Terminal

To run the TAF terminal, follow these steps:

  1. Connect the mobile device or start the server.
  2. Double-click the Terminal.jar file from the local folder.
  3. As a result, the taf terminal will connect to the specified server or mobile device, and the log details are displayed.
