Define Work Packages

After the work package is created, the test manager or test lead can map the features, test suites, and test cases with the target devices and testing environments. The combination of the test cases, test suites, or features with the target devices and environments are allocated with the resources based on the demand raised for the work package for a specific shift and date. The test manager or test lead can allocate test cases to a resource, after the resources are reserved for a shift. Before allocating test cases to a resource, select the shift and date on which test cases are completed.

 

Assign test cases and test suites

To create test case and environment combinations, follow these steps:

  1. Click Plan > Test > Work Packages. On the Work Packages pane, expand the product > build, and then select the work package.
  2. On the Work Package Execution page, click the Plan tab, and click the Feature, TestSuite, or Test Case button.
  3. Based on the selected option, the corresponding feature, test suite, or test case list will be displayed.
  4. Select the feature, test suites, or test cases, and click the Environment Combinations button.
  5. On the Select Environment Combination(s) page, click the Target Device Mapping () icon, and drag and drop the devices from the Available Devices list to the Mapped Devices list.
  6. Select the corresponding environment combinations, and close the page.
  7. Click the Allocate Test Cases tab to view the list of test cases along with the environment combinations.

Allocate resources for the test cases

To allocate resource for the test cases, follow the steps below:

  1. Click Plan > Test > Work Packages. On the Work Packages pane, expand the product > build, and then select the work package.
  2. On the Work Package Execution page, click the Allocate Test cases tab.
  3. Click the Feature, Test Suite, or Test Case button to list the available features, test suites, or test case in combination with the environment combinations.
  4. Select the execution date, shift, test lead name, and tester name for the listed feature, test suite or test case.
    Note: For Flex resource, the test lead and tester names will be available only when the resources are reserved for the specific work package on the specific date and shift. For more information, see Reserve resources.

To allocate a resource to multiple test cases, follow the steps below:

  1. Click Plan > Test > Work Packages. On the Work Packages pane, expand the product > build, and then select the work package.
  2. On the Work Package Execution page, click the Allocate Test cases tab.
  3. Select the Planned execution date, shift, test lead name, and tester name.
    Note: For Flex resource, the test lead and tester names will be available only when the resources are reserved for the specific work package on the specific date and shift. For more information, see Reserve resources.
  4. Click the Feature, Test Suite, or Test Case button, and select the corresponding features, test suites, or test cases.
  5. Click the Save button to allocate the selected resources for the features, test suites, or test cases.
