Frequently Asked Questions

This topic contains answers to frequently asked questions about Integrated Life Cycle Management (ILCM).

 

1. What is My Status page?

This page displays the execution status of work packages, which are allocated to you.

2. What is Test Run Plan?

Test run plan is a template, which is set with the Features, Test Suites, Test Case, Environments, and Devices. Based on the template, you can create manual or automated work packages. For more information, see Product Management.

3. What are the prerequisites to execute the Test Run Plan to automatically complete a work package?

In order, to execute the test run plan for web application, make sure the following prerequisites are in place:

  1. The terminal from the which the web application to be tested, is active.
  2. The URL of the web application to be tested, is included in the Web Application URL field of the product version.
  3. The file location of the testing script is included in the Script File Location field of the Test Suite.
    Note: The file location of the testing script must be in the following format: (file:///d:/TestScripts/file_name.zip)
4. How to include Test Factory Core Resource on the Flex Testing Team tab in the Product Management page?

The test factory core resource list will be updated based on the core resource mapped to the engagement of the selected product. Only Admin of the engagement can add core resource to the engagement. For more information about mapping the core resource to the engagement, Create a Test Factory.

5. Is it possible to change the default reset password?

Yes, it is possible to change the default reset password. In order to change the default reset password, you must have the administrator privileges to access the iLCM application folder. To change the default reset password, follow these steps:

  1. In the iLCM application folder (Server), open the \WEB-INF\classes\app.properties file in Notepad.
  2. Modify the DEFAULT_PASSWORD field with the required password.
  3. Save and close the app.properties file, and then restart the server.
