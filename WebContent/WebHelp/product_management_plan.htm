Products

A product is an application to be tested for bugs and accuracy of business needs in a test factory. Development stage of the product is marked by its version and build number. The version numbers are package with new features and bug fixes. The build numbers are packages with a minimal fixes for the bugs and defects. You can assign the test suites, test cases, de-coupling category, testing devices, features, and testing environment to the product. You can also define or set test suites, and test run plans to product versions.

Products

In this tab, you can define product and its hierarchy such as version, and build. Also, you can define a user role for this product. For example, a user can be Test Manager for a test factory, but he can act as Test Lead or Tester for a specific product. You can set the user role of the test manager as a test lead or tester, such that the user can act as Test Lead or Tester only for this product.

Product Summary

This tab displays the overall summary of the selected product. This tab includes three sections as Product summary, Operational summary, and Resource Summary. In the product summary section, the name, customer name, product mode, engagement to which the product belongs to, number of product versions, product builds, workpackage, features, test cases, test suites, and environments are displayed. In the operational summary section, it displays number of active work packages, and unmapped test cases. In the Resource summary section, it displays number of available test manager, test leads, testers, and mapped resource pools for the product.

Test Run Plan

Test run plan is a template, where you can create work packages along with the test suites, device, and environment combinations. These work packages can be executed either manually or automatically. You can create test run plans to execute the test cases available in the test suites in a target device. Before you create and execute the test run plan, make sure the test automation framework and other corresponding parameters are set.

In this tab, you can set up test suites, target device, execution type, and schedule date to execute the test cases. To create and execute the test run plan, follow the steps:

  1. On the Test Run Plan tab, click the Add Test Run Plan button, enter the name and description, and select the execution type either Manual or Automated option, schedule date and Test Tool Master.

  2. Click the Submit button to create a test run plan, and then click the Continue button.

  3. On the Test Suites tab, click the TestSuite tab, and select  test suites for execution.
    Note: Currently, iLCM supports  TestSuite only.

  4. Click the Continue button, and on the Test Beds tab, map the target devices for the environment combination.

  5. Select the test beds, and click the Continue button.

  6. On the Test Beds Distribution tab, the list of test cases and test bed combination is displayed, and click the Continue button.

  7. On the Execute tab, in the Product Builds field, select the build number, and then click the Execute button.

  8. Click the Execute button to execute the test cases in the test suite on the target device.
