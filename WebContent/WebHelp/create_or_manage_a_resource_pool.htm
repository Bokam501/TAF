Create a Resource Pool

In a test center, each test factory is mapped to a resource pool. A resource pool contains group of resources with various roles and responsibilities. The Admin or resource manager of the test factory can create and manage resource profiles for the resource pool.

 

Create a resource pool for a test center

To create a resource pool for a test factory center using the Admin role, follow these steps:

  1. If the user is set with the Admin role for the test factory center, click Admin > Manage Test Center.
  2. On the Manage Test Center page, click the user icon to display the existing the resource pool.
  3. On the Resource Pools pane, click the Add button, and enter the resource pool details such as Resource Pool Name, Display Name of the resource pool, City, State and Country where the resource pool exits, and the TestFactoryLab for which the resource pool is assigned to.
  4. Click the Save button to create the resource pool for the selected test factory center.
  5. If required, clear the Status checkbox to change the state of the resource pool to inactive.
 
— OR —
 

To create a resource pool for a test factory center using the Resource Manager role, follow these steps:

  1. Login as the Resource Manager role for a test center, click Resource Pool Management > Resource Pool.
  2. On the Test Centers pane, select the corresponding test factory center for which the resource pool is created for.
  3. Repeat steps 3 through 5 from the above task.

Map a resource pool to a test factory

To map a resource pool to a test center, follow these steps:

  1. Login as the Resource Manager role for a test center, click Resource Pool Management > Resource Pool.
  2. On the Test Centers pane, select the corresponding resource pool, and on the Resource Pool tab, click the Add/Edit Test Factory () icon.
  3. On the Add/Edit Test Factory pane, click the Add button, and select the Test Center and Test Factory from the drop-down lists.
  4. Click the Save button to map the test factory to the resource pool.

Create resource profiles for a resource pool

To create a resource for a resource pool using the Admin role, follow these steps:

  1. Click Admin > Manage Application Users.
  2. On the Manage Users pane, select the test factory center, and the Resource Pool. As a result, the User pane displays the list of existing users.
  3. Optional: On the User pane, click the Choose File button to import the excel sheet, which includes user details.
  4. On the User pane, click the Add button, and enter details of the resource such as First Name, Last Name, Middle Name, Login ID, Password, and Display Name.
  5. By default, a unique user Login ID is created for the user. If required, the admin can change the user login ID.
  6. In the User Type drop-down list, select the FTE, Contract Resource, or Flexi option.
  7. In the User Role drop-down list, select the Admin, Test Factory Manager, Test Manager, Test Lead, Tester, Program Manager, or Resource Manager option.
  8. In the Resource Pool drop-down list, select the corresponding resource pool for which the resource is allocated, and in the Vendor drop-down list, select the corresponding vendor.
  9. Click the Save button to create the resource and assign him to the corresponding resource pool.
  10. If required, clear the Status checkbox to mark the resource account is inactive.
  11. Optional, select the All, Active or Inactive option in the Status drop-down list to filter the resource account in the selected status.
  12. If required, click the Delete button to delete the resource account.
    Note: The user profile is deleted, but the user work details are available in the database. Also, the admin or resource manager can't create the same user ID again.
 
— OR —
 

To create a resource for a resource pool using the Resource Manager role, follow these steps:

  1. Click the Resource Pool Management > Manage Resource Pool Resources. As a result, the Manage Resource Pool Resource page displays Manage Users and User panes.
  2. On the Manage Users pane, select the corresponding test factory center and the Resource Pool. As a result, the User pane displays the list of existing users for the Resource pool.
  3. Repeat Steps 3 through 12 from the above task.
