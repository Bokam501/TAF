Manage Product Attributes

Product include test cases, test suites, features, decoupling category to categories test cases, testing environments and devices, change requests, risk, and automation scripts. The test cases are mapped to multiple features to ensure that common functionality in the features can be tested. Decoupling category is used to categories the test cases based on a user type. Test cases status can be tracked based on the feature, decoupling category, and version.

 

Manage Test Cases

Test cases for testing the product can be added to the products along with the test steps. You can add the test cases to a product in the following ways:

  • Manually add the test cases along with test steps
  • Import test cases from an excel sheet
  • Import test cases from test management system like HPQC
  • Automated test cases generation after executing the test scripts in test suites

To manually add the test cases, follow these steps:

  1. Click Setup > Product Management. As a result, the Manage Products page is displayed.
  2. On the Products pane, select the Engagement > Product, and then on the Test Cases tab, click Add.
  3. In the Add page, enter the test case code, test case name, and description about the test case.
  4. Select either manual or automated execution type.
  5. For automated test case execution, enter the script location and script name.
  6. In the Priority drop-down list, select the level of priority for the test case. Also, in the Test Case Type drop-down list, select the type of the test case.
  7. In the Feature drop-down list, select a feature to map the test case with the feature to ensure that the test case is available for that feature. You can create a feature and map the test cases with it in the Features tab.
  8. In the Decoupling Categories drop-down list, select a decoupling category to map the test case. You can create and map a decoupling category with the test cases in the Decoupling Categories tab.
  9. Click Save to create the test case.

To add test steps for the test case, follow these steps:

  1. Click the Test Step List ( ) icon, which is before the Test Case ID field, and on the Test Steps pane, click Add.
  2. In the Add page, enter the test step name, description for the test step, test step input and expected output, test step code for the test case.
  3. Click Save to include the test step for the test case.

 

To import test cases from an excel sheet, follow these steps:

  1. Click Setup > Product Management. As a result, the Manage Products page is displayed.
  2. On the Products pane, select the Engagement > Product, and then on the Test Cases tab, click the TESTCASE TEMPLATE link to download the template.
  3. Update the template with the test cases, test steps, and feature details.
  4. Click TestCases or TestSteps to upload the test cases and test steps.

To import test cases from test management system like HPQC, follow these steps:

  1. Click Setup > Product Management. On the Products pane, select the Engagement > Product1.
  2. On the Manage Products page, click the Tool Integration tab.
  3. On the Test Management System tab, click the Add button. As a result, the Add page is displayed.
  4. In the Title field, enter the test management system name i.e. HPQC, and description, and enter the cron expression. The Cron expression is a string which triggers the scheduler to execute the test run plans. For more details, Cron Expression.
  5. In the Test System drop-down list, select the test management system as HPQC, and in the System Type drop-down list, select the either 32-bit or 64-bit OS.
  6. Select the Is Primary check box to set the configured management system as primary system for test cases import.
  7. In the Version field, enter the test management system version number, and in the Connection Uri field, enter the url of the test management system say “http://10.108.57.17:8080/qcbin”.
  8. Enter the username, password, and domain, which is the domain name of the management system, and then click the Save button. TestSetFolder Filter (ID), TestSetFolder Filter(NAME), TestCase Filter(ID), and Resource filter (ID) are place holder fields for future reference.

Before performing the uploading or downloading operation, you must establish the connection between the HPQC and iLCM. For example, the HPQC has a domain i.e. default, and a project i.e. product1. Add the entity details to import the test cases from the HPQC. To add the entity details, follow these steps:

  1. On the Test Management System tab, click the Test Management System Mapping () icon, and then on the Test Management System Mapping pane, click the Add button.
  2. In the Mapping Entity drop-down list, select one of the following options:
    • Product (Select this option to view the list of products available in the HPQC)
    • Product Version
    • Test Suite
  3. In the Entity Name drop-down list, select the entity name i.e. Product 1. The test case for the selected product name will be imported from the test management system.
  4. In the Mapping Value field, enter the mapping value for the product name i.e. Product. Here the Mapping value indicates the project name in HPQC system.

 

 

Manage Test Suites

Test suites are available only for a product version. Before mapping the test cases to a test suite, you must map the available test cases to a product version in the Product Version Test Case Mapping page.

 

To map a test case to a product version, follow these steps:

  1. Click Setup > Product Version Test Case Mapping.
  2. On the Products pane, select an Engagement > Product. As a result, the Product Version Test Case Mapping tab displays list of available test cases and product versions check boxes.
  3. Select the product version check box that corresponds to a required test case to be mapped.

To create a test suite and map the test cases to it, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and on the Test Suites tab, click the Add button.
  3. Enter the test suite name and execution type.
  4. Click the View Test Cases () icon to view the list of mapped test cases and test suites with the test suite.
  5. Click the Test Case Mapping () icon, and drag and drop the test cases and test suites from the Available Test Cases/Test Suites list to the Mapped Test Cases/Test Suites list.

-OR-

 

To create a test suite for automated testing (Test Run Plan), follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product > Version, and on the Test Suites tab, click the Add button.
  3. Enter the test suite name, and select the Script Type.
  4. Enter the script file location, select the Execution Type as either Automated or Manual, and then click the Save button.

Manage Features

To map a test case to a product feature, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product, and on the Features tab, click the Add button.
  3. Enter the feature name and code, and then select the parent feature.
  4. Click the Save button to create a product feature.
  5. On the Feature pane, click the Mapped Test Cases () icon to view the list of mapped test cases.
  6. Click the Test Case Mapping () icon, and drag and drop the test cases from the Available Test Cases list to the Mapped Test Cases list.

Manage Decoupling Category

To map a test case to a decoupling category, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product, and on the Decoupling Category tab, click the Add button.
  3. Enter the decoupling category name, and then select the parent category and user type. If the newly added decoupling category is the parent category, then select the null option in the Parent Category drop-down list.
  4. Click the Save button to create a product decoupling category.
  5. On the Decoupling Category pane, click the Mapped Test Cases List () icon to view the list of mapped test cases.
  6. Click the Test Case Mapping () icon, and drag and drop the test cases from the Available Test Cases list to the Mapped Test Cases list.

Manage Environments

The Environment Group page allows to create testing environments and testing environment groups. The testing environment group is collection of testing environments. You can define the testing environment in which the product is to be tested. Note: Environment Group is available only for the Admin.

To create an environment group, follow these steps:

  1. Click the Admin > Environment Group, and click Add.
  2. Enter the group name such browser, description, and display name, and then click Save.
  3. Expand the Environment Category icon, and click Add.
  4. Enter the category name such Firefox, description, display name and parent category.

After creating the environment group and category, the same will be displayed in the product management page to create environment and its combination.

To create testing environment combination, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product, and on the Environment tab, click the Add button.
  3. Enter the testing environment name, and then select the environment group and category.
    Note: Environment Group name is defined in the Admin > Environment Group.
  4. Click Save to create a testing environment.
  5. Based on the added environments and environment category, the environment combination is created.
    For example, if Google Chrome is added under the Browser environment category, and the Smart phone and Tablet are added under the device environment category, then the possible environment combination are Google Chrome - Smart phone, and Google Chrome - Tablet.

Manage Device

The generic devices can be added automatically when connected to the terminal, or you can manually add the devices to the server. To add the devices to the devices list, follow these steps:

  1. Click the Devices > Device List, and click Mobile or Server.
  2. Click Add New Generic Devices, and in the Add Device page, enter the device name and description.
  3. Select the device status as Yes or No, and then the platform type.
  4. Select the device model master, and device type. Based on the selected device type, the Mobile or Server Device Details information will be displayed.
  5. In the Version field, enter the mobile device version number.
  6. Fill the device details. For server type devices, enter the host name, IP address, system type, and processor. For mobile type devices, enter the kernel number, build number, and device make master name.
  7. Click Create to create the device.

After creating the devices in the device list, you can map the device to the product for testing. To map the device to the product, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product, and on the Devices tab, click either Map Mobile or Map Server.
  3. In the Map Target Mobiles or Map Target Server window, drag and drop the mobile devices or servers available in the Mobile device or Server list to the Mapped Mobile Device or Mapped Server list.
  4. Click Mobile or Server in the type field to view the list of active or inactive devices.

Manage Change Request

To map change requests received for the product, follow these steps:

  1. Click Setup > Product Management.
  2. On the Products pane, select an Engagement > Product, and on the Change Request tab, click Add.
  3. On the Add page, enter the change request name and description.
  4. Select the owner name and execution priority for the change request.
  5. Click Save to create and map the change request for the product.

You can also upload the change request using the change request template. To upload the change request, follow these steps:

  1. Click Setup > Product Management. As a result, the Manage Products page is displayed.
  2. On the Products pane, select the Engagement > Product, and then on the Change Request tab, click the Change Request TEMPLATE link to download the template.
  3. Update the template with the change request details.
  4. Click Change Requests to upload the change request.
